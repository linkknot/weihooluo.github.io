macOS sierrra重启卡旋转菊花

发表于 2017-10-25 | 分类于 开发环境

背景

近期在尝试重启macbook pro的时候卡在旋转菊花的地方，系统久久不能重启，只能通过强制关机，然后开机才能进入系统，毕竟mac很少重启，所以这个问题比较迟发现

分析最近操作

  1. 难道做了什么骚操作导致系统文件的权限出现问题
  2. 最近一般就重新安装ffmpeg，为啥要重装？因为系统macOS更新后，用homeberw安装的ffmpeg在用的时候就出现无法加载包的问题

发现问题过程

  1. 第一个了解到的可能的问题是：系统文件的权限出现了问题

    可能的解决方式：有没有检查系统文件权限的工具

    1. 检测的工具有，但在macOS sierra上已经取消了，macOS sierra对系统文件的权限进行了保护
    2. 那就应该没系统文件权限的事了

  2. 系统更新后出现一个与软件的兼容问题，导致系统无法正常重启

    这个也是一个可能，因为出现过ffmpeg更新后不能使用，虽然原因不明

    暂且认为可能升级系统导致

    在后续的google中，大部分的问题都是指向系统文件权限，之后知乎上看到是系统的更新后的mac的某些服务与mac关机发生了冲突，导致无法关机，答主同样也是mysql服务无法停止

问题解决

  1. 尝试在偏好设置停止mysql服务，结果是卡死，无法关闭
  2. 通过ps -ax | grep mysql找出所有mysql的进程，并且把他们kill掉，mysql服务终于停了
  3. 卸载mysql，重新安装，之前是使用mysql安装包安装，之后型通过brew进行安装，较少不必要的问题
  4. 通过brew重新安装

卸载mysql

  1. 使用brew安装

    1. `brew uninstall mysql 或者 brew remove mysql

  2. 使用安装包安装

    到指定的目录下删除

为了保证下一次安装干净，建议还是检查一下以上的目录，包括

  • /usr/local/Cellar 里的mysql文件
  • /usr/local/var 里的mysql文件
  • /tmp 里的mysql.sock, mysql.sock.lock, my.cnf文件
  • pid文件和err文件都在/usr/local/var/mysql里确保删除了
  • brew安装的安装包存储在/usr/local/Library/Cache/Homebrew也可以一并删除
  • 执行brew cleanup

安装mysql

brew install mysql

安装的时候会出现各种选项，让你选择，按照你的实际使用选择就可以，brew 会提示你需要执行一条命令设置密码的，一切安装提示做就可以

完成安装后：

通过$ mysql.server start就能启动mysql

其他选项还有{start|stop|restart|reload|force-reload|status}
